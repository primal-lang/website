What is Primal?

Primal is:

  • Inspired in Lambda Calculus
  • Declarative
  • Functional
  • Esoteric
  • Simple
  • Interpreted
  • Interactive
  • Free
  • Open source
  • Minimalist
  • Lazy evaluated
  • Lightweight
  • Expression oriented

Although Primal is a general purpose language, it's mostly intended to be used as a tool for learning functional programming.

It is a programming language designed to be simple and easy to learn, primarily used for educational purposes. Primal aims to teach programming concepts without the complexity of more advanced languages. It emphasizes simplicity and minimal syntax, making it a good choice for beginners who are learning the fundamentals of programming.

Everything in Primal is a function definition Function can (and should) be composed Function cannot be overloaded (i.e. only one function with a given name can exist in a program) Currently there are no operators. Instead Primal offers a function with equivalent functionality Primal encourages to write short, very specific function to increase reusability Primal has a dynamic runtime type checking where: * User-Defined Types: Users cannot explicitly declare types for parameters, return values, or variables in the code. * Runtime Checking: The system performs type checks during execution to ensure that values match the expected types when passed to functions or operations. This approach allows for flexibility and ease of use but relies on runtime checks to catch type-related errors, which can impact performance and may not catch all errors until the code is executed. This allows for flexibility in coding but can lead to type-related errors that are only caught during execution. Unlike statically typed languages, where types are explicitly declared and checked at compile time, Primal determines types on the fly as the program runs. Function name format: [a-zA-Z]+[a-zA-Z_]+ Parameter name format: ??? Its goal is to offer a very simple and minimalist syntax for people that want to learn functional programming (or programming in general). Technical envelope: • Syntax rules • Lazy evaluated • Dynamic type system • Error Handling • Single threaded • Language constructs (no loops, only declarations) Primal is interpreted.
