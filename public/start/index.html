What is primal

Primal is:

  • Inspired in Lambda Calculus
  • Declarative
  • Functional
  • Interpreted
  • Interactive
  • Free
  • Open source
  • Minimalist
  • Lazy evaluated
  • Lightweight
  • Expression oriented

Goal

Its goal is to be used as a tool for learning functional programming (or programming in general). It's not intended to be used in production environments, but it can be used for small scripts.

It is a programming language designed to be simple and easy to learn, primarily used for educational purposes. Primal aims to teach programming concepts without the complexity of more advanced languages. It emphasizes simplicity and minimal syntax, making it a good choice for beginners who are learning the fundamentals of programming.

Everything in Primal is a function definition Function can (and should) be composed Function cannot be overloaded (i.e. only one function with a given name can exist in a program) Currently there are no operators. Instead Primal offers a function with equivalent functionality Primal encourages to write short, very specific function to increase reusability Primal has a dynamic runtime type checking where: * User-Defined Types: Users cannot explicitly declare types for parameters, return values, or variables in the code. * Runtime Checking: The system performs type checks during execution to ensure that values match the expected types when passed to functions or operations. This approach allows for flexibility and ease of use but relies on runtime checks to catch type-related errors, which can impact performance and may not catch all errors until the code is executed. This allows for flexibility in coding but can lead to type-related errors that are only caught during execution. Unlike statically typed languages, where types are explicitly declared and checked at compile time, Primal determines types on the fly as the program runs.
