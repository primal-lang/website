Reference

Primal has a rich set of core functions that provide essential support for developers. These functions are designed to be simple and easy to use, while also being powerful and flexible. They can be used to build complex applications and solve a wide range of problems.

They are divided in the following categories:

  • Control
  • Error
  • Debugging
  • Comparison
  • Arithmetic
  • Logic
  • String
  • Casting

Note: Primal doesn't support types yet, but in order to improve the readability of the documentation, each function's signature has the parameters and the return value typed (or marked as generic when it applies).

Control

If-else

Input: One conditional argument (a) and two arguments of any type (b and c). The conditional argument must evaluate to a boolean. The other two arguments must evaluate to the same type.

Output: If the condition is true, it returns the first argument. Otherwise it returns the second argument.

if(a: Boolean, b: T, c: T): T

Try-catch

Input: Two arguments, which must evaluate to the same type.

Output: It returns the first argument unless it evaluates to an error, in which case, it returns the second argument.

try(a: T, b: T): T

Error

Throw error

Input: An argument of any type.

Output: It returns an error wrapping that argument.

error(a: T): Error

Debugging

Debug

Input: An argument of any type.

Output: It writes that argument in the standard output and returns it.

debug(a: T): T

Comparison

Equality

Input: Two arguments of any type.

Output: True if they are equal. False otherwise.

Inequality

Input: Two arguments of any type.

Output: True if they are not equal. False otherwise.

Greater than

Less than

Input: Two numbers.

Output: True if the first number is less than the second number. False otherwise.

lt(a: Number, b: Number): Boolean

Greater than or equal

Input: Two numbers.

Output: True if the first number is greater than or equal to the second number. False otherwise.

ge(a: Number, b: Number): Boolean

Less than or equal

Input: Two numbers.

Output: True if the first number is less than or equal to the second number. False otherwise.

Logic

And

Input: Two boolean arguments.

Output: True if both arguments are true. False otherwise.

and(a: Boolean, b: Boolean): Boolean

Or

Input: Two boolean arguments.

Output: True if at least one argument is true. False otherwise.

or(a: Boolean, b: Boolean): Boolean

Xor

Input: Two boolean arguments.

Output: True if exactly one argument is true. False otherwise.

xor(a: Boolean, b: Boolean): Boolean

Not

Input: One boolean argument.

Output: True if the argument is false. False if the argument is true.

not(a: Boolean): Boolean
